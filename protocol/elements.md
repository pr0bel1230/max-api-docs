# Rich-элементы (elements[])

Поле `elements` в структуре сообщения содержит массив аннотаций
к тексту. Каждый элемент описывает интерактивный или форматированный
фрагмент текста.

## Формат

```json
{
  "type": "LINK",
  "from": 258,
  "length": 50,
  "attributes": {
    "url": "https://example.com/page"
  }
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `type` | string | Тип элемента |
| `from` | int | Смещение от начала `text` в символах |
| `length` | int | Длина фрагмента в символах |
| `attributes` | object | Атрибуты, специфичные для типа |

## Зачем нужны elements[]

Без elements[] клиент вынужден парсить текст регулярными выражениями
для поиска ссылок, упоминаний и эмодзи. Это ненадёжно: regex может
не совпасть с URL, сломаться на экранированных символах или найти
ложные срабатывания.

elements[] решает это — сервер сам размечает текст, указывая точные
позиции и типы элементов. Клиенту остаётся только применить
форматирование или интерактивность по этим координатам.

## Известные типы

### LINK — гиперссылка

Аннотирует фрагмент текста, который является кликабельной ссылкой.

```json
{
  "type": "LINK",
  "from": 258,
  "length": 50,
  "attributes": {
    "url": "https://priem.mai.ru/spec/programs/..."
  }
}
```

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `url` | string | Полный URL для перехода |

**Пример:** Если `text = "Подробнее на сайте https://mai.ru"`, то
LINK-элемент с `from=19, length=17, attributes.url="https://mai.ru"`
аннотирует URL внутри текста.

### Другие типы (предположительно)

На момент исследования встречен только `LINK`. Предположительно
поддерживаются:

| type | Описание | Предполагаемые attributes |
|------|----------|--------------------------|
| `EMOJI` | Кастомный эмодзи (не Unicode) | `emojiId`, `url` |
| `MENTION` | Упоминание пользователя | `userId` |
| `BOLD` | Жирное начертание | — |
| `ITALIC` | Курсив | — |
| `STRIKETHROUGH` | Зачёркнутый текст | — |

## Использование в запросах

При отправке или редактировании сообщения (MSG_SEND, MSG_EDIT)
поле `elements` обязательно. Для обычного текста без форматирования
передаётся пустой массив:

```json
"elements": []
```

Если передать элементы с форматированием, сервер проверит, что
`from` + `length` не выходят за границы текста, но точная валидация
зависит от типа элемента.
