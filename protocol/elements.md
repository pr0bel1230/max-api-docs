# Rich-элементы (elements[])

Поле `elements` в структуре сообщения содержит массив аннотаций
к тексту. Каждый элемент описывает форматированный или интерактивный
фрагмент текста: ссылку, жирное начертание, подчёркивание.

## Формат

```json
{
  "type": "LINK",
  "from": 258,
  "length": 50,
  "attributes": {
    "url": "https://example.com/page"
  }
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `type` | string | Тип элемента |
| `from` | int | Смещение от начала `text` в символах (0 = начало текста, может отсутствовать) |
| `length` | int | Длина фрагмента в символах |
| `attributes` | object | Атрибуты, специфичные для типа |

**Важно:** `from` может отсутствовать — в этом случае он считается равным 0.
Элементы могут перекрываться (например, `STRONG` + `UNDERLINE` на одном диапазоне).

## Где находятся elements[]

`elements[]` **не возвращается** в ответе на MSG_SEND и MSG_EDIT.
Он присутствует только в ответе GET_HISTORY и в push-уведомлениях
о новых сообщениях. Чтобы получить elements[], запроси историю
сообщений через опкод 49.

## Зачем нужны elements[]

Без elements[] клиент вынужден парсить текст регулярными выражениями
для поиска ссылок, упоминаний и форматирования. Это ненадёжно: regex может
не совпасть с URL, сломаться на экранированных символах или дать
ложные срабатывания.

elements[] решает это — сервер сам размечает текст, указывая точные
позиции и типы элементов. Клиенту остаётся только применить
форматирование или интерактивность по этим координатам.

## Известные типы

### LINK — гиперссылка

Аннотирует фрагмент текста, который является кликабельной ссылкой.

```json
{
  "type": "LINK",
  "from": 258,
  "length": 50,
  "attributes": {
    "url": "https://priem.mai.ru/spec/programs/..."
  }
}
```

| Атрибут | Тип | Описание |
|---------|-----|----------|
| `url` | string | Полный URL для перехода |

**Пример:** Если `text = "Подробнее на сайте https://mai.ru"`, то
LINK-элемент с `from=19, length=17, attributes.url="https://mai.ru"`
аннотирует URL внутри текста.

### STRONG — жирное начертание

Фрагмент текста с жирным начертанием (аналог `<strong>` или `<b>` в HTML).

```json
{
  "type": "STRONG",
  "from": 3,
  "length": 45
}
```

Не имеет attributes.

### EMPHASIZED — выделение (курсив)

Фрагмент текста с акцентированием (аналог `<em>` или `<i>` в HTML).

```json
{
  "type": "EMPHASIZED",
  "length": 7,
  "from": 183
}
```

Не имеет attributes.

### UNDERLINE — подчёркивание

Фрагмент текста с подчёркиванием.

```json
{
  "type": "UNDERLINE",
  "length": 56,
  "from": 51
}
```

Не имеет attributes. Часто перекрывается с `STRONG` на том же диапазоне
(текст одновременно жирный и подчёркнутый).

## Неподтверждённые типы

На момент исследования не встречены, но могут существовать:

| type | Описание | Предполагаемые attributes |
|------|----------|--------------------------|
| `EMOJI` | Кастомный эмодзи (не Unicode) | `emojiId`, `url` |
| `MENTION` | Упоминание пользователя | `userId` |
| `STRIKETHROUGH` | Зачёркнутый текст | — |

**Примечание:** Ранее предполагаемые типы `BOLD` и `ITALIC` в реальном
трафике не обнаружены. Вместо них используются `STRONG` и `EMPHASIZED`
соответственно.

## Перекрытие элементов

Один фрагмент текста может одновременно иметь несколько типов
форматирования. В таких случаях для одного диапазона создаётся
несколько элементов:

```json
[
  {"type": "STRONG", "length": 42},
  {"type": "STRONG", "length": 56, "from": 51},
  {"type": "UNDERLINE", "length": 56, "from": 51, "attributes": {"url": "..."}},
  {"type": "LINK", "length": 56, "from": 51}
]
```

В этом примере текст с позиции 51 длиной 56 символов является
одновременно жирным (`STRONG`), подчёркнутым (`UNDERLINE`) и ссылкой
(`LINK`).

## Использование в запросах

При отправке или редактировании сообщения (MSG_SEND, MSG_EDIT)
поле `elements` обязательно. Для обычного текста без форматирования
передаётся пустой массив:

```json
"elements": []
```

Если передать элементы с форматированием, сервер проверит, что
`from` + `length` не выходят за границы текста, но точная валидация
зависит от типа элемента. На практике сервер поддерживает полный цикл:
можно отправить элементы в MSG_SEND и получить их же в GET_HISTORY.

## Сводка

| Тип | Описание | Атрибуты | Где встречается |
|-----|----------|----------|-----------------|
| `LINK` | Гиперссылка | `url` | Каналы, диалоги |
| `STRONG` | Жирное начертание | — | Каналы (посты с разметкой) |
| `EMPHASIZED` | Курсив | — | Каналы |
| `UNDERLINE` | Подчёркивание | — | Каналы (часто с STRONG) |
