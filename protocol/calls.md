# Звонки: голосовые и видеозвонки

MAX поддерживает голосовые и видеозвонки между пользователями.
API звонков задокументирован частично — основные опкоды работают,
но требуют специфических payload.

## Структура звонка

В истории сообщений звонки представлены как сообщения с вложением типа `CALL`:

```json
{
  "sender": 3260455,
  "id": "115775937927062388",
  "time": 1766600615342,
  "text": "",
  "type": "USER",
  "attaches": [
    {
      "duration": 926791,
      "conversationId": "8daa66c0-04c1-4824-820b-67f92e6fd29d",
      "hangupType": "HUNGUP",
      "_type": "CALL",
      "callType": "AUDIO",
      "contactIds": [6236697]
    }
  ]
}
```

### Поля вложения CALL

| Поле | Тип | Описание |
|------|-----|----------|
| `_type` | string | Всегда `"CALL"` |
| `callType` | string | Тип звонка: `AUDIO` или `VIDEO` |
| `conversationId` | string | UUID звонка |
| `duration` | int | Длительность в миллисекундах |
| `hangupType` | string | Причина завершения (см. ниже) |
| `contactIds` | array[int] | ID участников звонка |

### Hangup types

| Значение | Описание |
|----------|----------|
| `HUNGUP` | Завершён (нормальное завершение) |
| `REJECTED` | Отклонён |
| `CANCELED` | Отменён (не дождались ответа) |
| `MISSED` | Пропущен |
| `BUSDY` | Занято (опечатка сервера — должно быть `BUSY`) |

## CALL_HISTORY (opcode 79)

Получение истории звонков в чате.

### Запрос

```json
{
  "chatId": 7268926,
  "count": 10
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatId` | int | ID чата (обязательно) |
| `count` | int | Количество записей (опционально, по умолчанию — все) |

### Ответ

```json
{
  "hasMore": true,
  "history": [
    {
      "chatId": 7268926,
      "message": { ... },
      "chatType": "DIALOG"
    }
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `hasMore` | bool | Есть ли ещё записи |
| `history` | array | Массив записей звонков |
| `history[].chatId` | int | ID чата |
| `history[].message` | object | Сообщение с вложением CALL |
| `history[].chatType` | string | Тип чата |

### Особенности

- **hasMore** = true означает, что есть более старые звонки (пагинация)
- Каждая запись содержит полный объект сообщения с attaches типа CALL
- callType: AUDIO или VIDEO
- duration = 0 означает пропущенный/отклонённый звонок

## CALL_EDIT (opcode 69)

Редактирование звонка (например, отключение звука, завершение).

### Запрос

```json
{
  "conversationId": "8daa66c0-04c1-4824-820b-67f92e6fd29d",
  "muteAudio": true,
  "muteVideo": false
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `conversationId` | string | UUID звонка |
| `muteAudio` | bool | Отключить микрофон |
| `muteVideo` | bool | Отключить видео |
| `muteScreenSharing` | bool | Отключить демонстрацию экрана |
| `onlyAdminCanRecord` | bool | Только админ может записывать |
| `waitingHall` | bool | Зал ожидания |
| `waitForAdmin` | bool | Ждать администратора |
| `closed` | bool | Завершить звонок |

### Внутренняя реализация

Судя по сообщению об ошибке, сервер обрабатывает через:
```
CallsServiceImpl.editCall(conversationId, userId, Mutes{...})
```

**Примечание:** на момент исследования опкод возвращает ошибку
валидации при попытке редактирования завершённого звонка
(`"No call <conversationId>"`). Требуется активный звонок.

## CALL_START (opcode 78)

Инициализация нового звонка.

### Запрос

```json
{
  "isVideo": false,
  "conversationId": "uuid",
  "chatId": 7268926,
  "internalParams": {}
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `isVideo` | bool | `true` — видеозвонок, `false` — аудиозвонок |
| `conversationId` | string | UUID новой конференции |
| `chatId` | int | ID чата |
| `internalParams` | object | Внутренние параметры звонка (обязательно) |

**Примечание:** на момент исследования не удалось найти полный payload
для успешного старта звонка. Требуется экспериментальное исследование
с активным клиентом.
