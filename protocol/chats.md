# Чаты: получение списка и управление

## GET_CHATS (opcode 53)

Получение списка чатов.

### Запрос

```json
{
  "count": 100,
  "marker": 0
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `count` | int | Количество чатов |
| `marker` | int | Timestamp (UNIX, мс). Возвращает чаты, изменённые ранее этого времени |

### Ответ

```json
{
  "chats": [
    {
      "id": 7268926,
      "type": "DIALOG",
      "title": "Имя собеседника",
      "lastMessage": {
        "text": "Последнее сообщение",
        "time": 1734567890123,
        "sender": 3260455
      },
      "newMessages": 5,
      "members": [
        {"id": 3260455, "role": "OWNER"},
        {"id": 3260456, "role": "MEMBER"}
      ],
      "picture": {
        "url": "https://...",
        "base": "..."
      },
      "createdTime": 1700000000000
    }
  ],
  "marker": 1764604380342
}
```

### Поля чата

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | ID чата |
| `type` | string | `DIALOG`, `CHAT`, `CHANNEL` |
| `title` | string | Название чата / имя собеседника |
| `lastMessage` | object | Последнее сообщение |
| `newMessages` | int | Количество непрочитанных |
| `members` | array | Участники чата |
| `picture` | object | Аватар чата |
| `createdTime` | int | Timestamp создания |

### Альтернативный источник: LOGIN

Список чатов также доступен в ответе LOGIN (opcode 19) — поле `chats`.
Если `chatsSync` и `chatsCount` установлены в 100, чаты придут
сразу при авторизации.

## GET_STATS (opcode 74)

Получение статистики чата.

### Запрос

```json
{
  "chatId": 7268926
}
```

### Ответ

```json
{
  "stats": {}
}
```

На момент исследования ответ содержал пустой объект `stats`.
Возможно, заполняется после накопления данных.

## CHAT_OPERATION (opcode 77)

Операции над чатами: закрепление, архивирование, отключение уведомлений.

### Запрос

```json
{
  "operation": "pin",
  "chatId": 7268926,
  "userIds": [3260455]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `operation` | string | Тип операции: `pin`, `archive`, `mute`, `read` |
| `chatId` | int | ID чата |
| `userIds` | array[int] | ID пользователей (обязательно) |

### Ответ

`cmd=1` (пустой объект `{}`).

### Операции

| Значение | Описание |
|----------|----------|
| `pin` | Закрепить чат |
| `archive` | Архивировать чат |
| `mute` | Отключить уведомления |
| `read` | Отметить как прочитанное |

**Примечание:** поле `userIds` обязательно — без него сервер возвращает
ошибку `"userIds can't be empty"`. Операции применяются к чату для
указанных пользователей.

## IMAGE_UPLOAD_URL (opcode 80)

Получение URL для загрузки изображений.

### Запрос

```json
{
  "count": 1
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `count` | int | Количество URL (опционально) |
| `photoIds` | array[string] | ID фотографий (опционально) |

### Ответ

```json
{
  "url": "https://iu.oneme.ru/uploadImage?apiToken=...&photoIds=..."
}
```

Возвращает URL для загрузки изображений на сервер MAX (`iu.oneme.ru`).
Каждый запрос генерирует новый `apiToken`.

## Типы чатов

| Тип | Описание | Возвращается в |
|-----|----------|----------------|
| `DIALOG` | Личный диалог между двумя пользователями | все endpoints |
| `CHAT` | Групповой чат (3+ участников) | все endpoints |
| `CHANNEL` | Канал (только для чтения для подписчиков) | все endpoints |

## Работа с чатами

- ID чата можно получить из `id` поля в объекте чата
- Для личных чатов `title` содержит имя собеседника
- `type` определяет, доступны ли групповые операции
- **Пагинация:** `marker` — это **timestamp** (UNIX, мс). Запрос
  возвращает чаты, у которых `lastEventTime < marker`, отсортированные
  по убыванию. В ответе поле `marker` содержит timestamp самого старого
  чата в выборке — его можно передать в следующий запрос для получения
  более ранних чатов.
- `marker=0` возвращает пустой список (`chats: []`). Для получения всех
  чатов используйте `marker = текущий_timestamp` (обычно передают
  `int(time.time() * 1000)`).
- Альтернативный источник списка чатов — ответ LOGIN (opcode 19) с
  параметром `chatsCount` и `chatsSync`.

## GET_CHATS_BULK (opcode 48)

Получение информации о нескольких чатах по их ID. Используется
при загрузке списка чатов и открытии папок.

### Запрос

```json
{
  "chatIds": [
    -72842705805946,
    7268926,
    0
  ]
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatIds` | array[int] | ID чатов. `[0]` как sentinel — получить все чаты |

### Ответ

```json
{
  "chats": [
    {
      "owner": 3260455,
      "joinTime": 1,
      "created": 1,
      "lastMessage": {
        "sender": 3260455,
        "id": "116590570569941100",
        "time": 1779030923003,
        "text": "",
        "type": "USER",
        "attaches": []
      },
      "type": "DIALOG",
      "modified": 1779355533793,
      "lastEventTime": 1779355533793,
      "id": 7268926,
      "status": "ACTIVE",
      "participants": {
        "3260455": 1779355533793
      }
    }
  ]
}
```

### Особенности

- `chatIds: [0]` возвращает все чаты (включая системный чат с id=0)
- Ответ содержит полные объекты чатов, как GET_CHAT_INFO (61)
- Вызывается при открытии папки или загрузке полного списка чатов
- Заменяет множественные вызовы GET_CHAT_INFO

## CHAT_SUBSCRIBE (opcode 75)

Подписка на события чата. Клиент вызывает при входе/выходе из чата.

### Запрос

```json
{
  "chatId": -72842705805946,
  "subscribe": true
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatId` | int | ID чата |
| `subscribe` | bool | `true` — подписаться на события, `false` — отписаться |

### Ответ

`cmd=1` (ACK, payload = null).

### Особенности

- Вызывается при каждом заходе в чат (`subscribe: true`) и выходе (`subscribe: false`)
- После подписки клиент начинает получать push-уведомления для этого чата
- Без подписки опкоды 128 (NOTIF_MSG_EDIT), 129 (NOTIF_TYPING), 156 (NOTIF_REACTION)
  и другие уведомления по чату могут не приходить
- **Обязательна** для получения push 137 (NOTIF_INCOMING_CALL) — без подписки
  на чат входящий звонок не будет доставлен

## CHAT_READ (opcode 50)

Отметка сообщений как прочитанных или непрочитанных.

### Запрос

```json
{
  "type": "READ_MESSAGE",
  "chatId": 7268926,
  "messageId": "116765779164748382",
  "mark": 1781704393993
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `type` | string | `"READ_MESSAGE"` или `"SET_AS_UNREAD"` |
| `chatId` | int | ID чата |
| `messageId` | string | ID сообщения (только при `READ_MESSAGE`) |
| `mark` | int | Timestamp отметки (миллисекунды) |

### Ответ

```json
{
  "unread": 1,
  "mark": 1781704393991
}
```

- `unread` — новое количество непрочитанных сообщений
- `mark` — подтверждённый timestamp отметки

### Отличие от CHAT_ACTIVITY (92)

CHAT_ACTIVITY (92) запрашивает историю чата за период и **попутно**
устанавливает водяной знак. CHAT_READ (50) — целенаправленная операция
для отметки о прочтении.

## VOID (opcode 55)

Заглушка. Возвращает `cmd=1` с пустым payload без выполнения каких-либо
действий. Назначение не установлено — возможно, heartbeat для проверки
активности чата или зарезервированный опкод для будущих функций.

### Запрос

```json
{
  "chatId": 7268926
}
```

### Ответ

`cmd=1` (ACK, payload = null).

## GET_CHAT_INFO (opcode 61)

Получение полной информации о чате.

### Запрос

```json
{
  "chatId": 7268926
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatId` | int | ID чата (обязательно) |

### Ответ

```json
{
  "chat": {
    "id": 7268926,
    "type": "DIALOG",
    "status": "ACTIVE",
    "owner": 3260455,
    "participants": {
      "6236697": 1781688781664,
      "3260455": 1781697221724
    },
    "lastMessage": {
      "id": "116765309122929651",
      "text": "последнее сообщение",
      "sender": 3260455
    },
    "joinTime": 1,
    "created": 1,
    "modified": 1781686441819,
    "lastEventTime": 1781686441819,
    "cid": 7268926
  }
}
```

### Поля chat

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int | ID чата |
| `type` | string | `DIALOG`, `GROUP`, `CHANNEL` |
| `status` | string | `ACTIVE`, `ARCHIVED` |
| `owner` | int | ID владельца чата |
| `participants` | object | `{userId: lastReadTimestamp}` |
| `lastMessage` | object | Последнее сообщение в чате |
| `joinTime` | int | Timestamp вступления |
| `created` | int | Timestamp создания чата |
| `modified` | int | Timestamp последнего изменения |
| `lastEventTime` | int | Timestamp последнего события |

### Особенности

- **participants** — значения — timestamp последнего прочтения в миллисекундах
- **joinTime/created** равны 1 если чат создан при миграции со старой версии
- Эквивалентен GET_HISTORY (49) с `{getChat: true}`, но без сообщений
- Дополнительные поля в запросе (например, `emoji`) игнорируются

## CHAT_ACTIVITY (opcode 92)

Получение информации о чате за временной диапазон.
Ранее ошибочно считался MSG_DELETE_RANGE.

### Запрос

```json
{
  "chatId": 7268926,
  "startTime": 1777393633877,
  "endTime": 1781697221724
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatId` | int | ID чата (обязательно) |
| `startTime` | int | Начало временного диапазона (мс) |
| `endTime` | int | Конец временного диапазона (мс) |

### Ответ

Возвращает полный объект чата (как GET_CHAT_INFO, 61) за указанный
временной диапазон:

```json
{
  "chat": {
    "owner": 3260455,
    "id": 7268926,
    "type": "DIALOG",
    "status": "ACTIVE",
    "participants": { ... },
    "lastMessage": { ... }
  }
}
```

### Особенности

- **messageIds** в запросе не поддерживаются — сервер ожидает `endTime`
- `startTime = 0` вызывает ошибку `"Key: Queue is not configured"`
- Требуются оба поля: `startTime` и `endTime`
- Ответ идентичен GET_CHAT_INFO (61) — возможно, это один и тот же
  endpoint с разными фильтрами
- **⚠️ ПОБОЧНЫЙ ЭФФЕКТ: УСТАНОВКА WATERMARK:** Успешный вызов CHAT_ACTIVITY (92)
  устанавливает на сервере внутреннюю метку `lastReadTimestamp` («водяной знак») —
  отметку о том, что клиент «прочитал» историю до этой точки.
  **Само по себе это безвредно** — 92 не удаляет сообщения (read-only).
  **ОПАСНОСТЬ:** Если сразу после 92 вызвать **MSG_DELETE (66) с `forMe: false`**,
  сервер удалит **ВСЕ сообщения до этой метки**, а не только указанные в `messageIds`.
  Подробнее в [messaging.md#msg_delete-opcode-66](messaging.md#msg_delete-opcode-66).

## CHAT_SHOW (opcode 86)

Управление отображением чата (показать/скрыть).

### Запрос

```json
{
  "chatId": 7268926,
  "show": true
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatId` | int | ID чата (обязательно) |
| `show` | bool | `true` — показать, `false` — скрыть |

### Ответ

`cmd=1` (ACK, payload = null при `show: true`).

### Особенности

- Поле `show` ожидает **boolean**, не строку
- `show: false` возвращает ошибку `"Key: chat.not-found"` (вероятно,
  требует дополнительных полей для скрытия)
- `show: true` без `chatId` возвращает `"Field requirement failed: chatId"`
- Назначение точно не установлено — возможно, управление видимостью
  в списке чатов или фильтрация

### Поля папки

| Поле | Тип | Описание |

| sourceId | Источник |
|----------|----------|
| 1 | Внутренний источник MAX |
| 479 | Внешнее приложение (Сферум, appId: 2340831) |

### Значения filters

| filters | Значение |
|---------|----------|
| `[]` | Все чаты |
| `[0]` | Непрочитанные |
| `[2]` | Каналы |
| `[13]` | Приложения (уточняется через `filterSubjects`) |

### Структура widget

| Поле | Тип | Описание |
|------|-----|----------|
| `appId` | int | ID приложения |
| `id` | string | UUID виджета |
| `name` | string | Отображаемое название |
| `description` | string | Описание |
| `iconUrl` | string | URL иконки |
| `background` | string | Цвет фона (hex) |
| `startParam` | string | Параметр запуска (пустая строка = главная) |

### Особенности

- **folderSync** в запросе: `0` = полная загрузка, timestamp из ответа
  = дельта-синхронизация (при повторном запросе с timestamp сервер
  возвращает только изменения)
- **chatId** может быть любым ID чата, а также `0` — во всех случаях
  возвращается одинаковый результат (список папок текущего пользователя)
- **Папки** — пользователь может создать несколько папок; `foldersOrder`
  определяет их порядок
- **include** содержит числовые ID чатов, входящих в данную папку
- **filterSubjects** — уточнение фильтра приложений: `{"13": [appId]}`
- **elements** — анимированные элементы оформления папки (стикеры/анимодзи)
- **widgets** — мини-приложения внутри папки (только для sourceId=479)
- Папки не могут быть получены с пустым payload — требуется chatId
