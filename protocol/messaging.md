# Сообщения: отправка, получение, редактирование, удаление

Управление сообщениями — основной функционал MAX API.
Все операции над сообщениями используют отдельные опкоды.

## Структура сообщения

Сообщение в MAX имеет следующую структуру (в ответах сервера):

```json
{
  "id": 116762160362694583,
  "time": 17596280448116,
  "type": "USER",
  "sender": 3260455,
  "cid": 35,
  "text": "текст сообщения",
  "elements": [],
  "attaches": [],
  "reactionInfo": {}
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `id` | int/number | Уникальный ID сообщения |
| `time` | int | Unix timestamp в миллисекундах |
| `type` | string | Тип: `USER`, `SERVICE`, `SYSTEM` |
| `sender` | int | ID отправителя (user_id) |
| `cid` | int | Client ID — идентификатор для дедупликации |
| `text` | string | Текст сообщения |
| `elements` | array | Rich-элементы (форматирование, ссылки) |
| `attaches` | array | Вложения (см. [files.md](files.md)) |
| `reactionInfo` | object | Реакции на сообщение |
| `deleted` | bool | Присутствует только у сервисных уведомлений |

**Примечание:** `id` может передаваться как число или как строка
в разных контекстах. При сравнении лучше приводить к строке.

## MSG_SEND (opcode 64)

Отправка нового сообщения в чат.

### Запрос

```json
{
  "chatId": 7268926,
  "message": {
    "text": "Привет!",
    "cid": 1734567890123,
    "elements": [],
    "attaches": []
  },
  "notify": true
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatId` | int | ID чата |
| `message.text` | string | Текст сообщения |
| `message.cid` | int | Client ID — `int(time.time() * 1000)` или произвольный |
| `message.elements` | array | Rich-элементы (пустой массив = обычный текст) |
| `message.attaches` | array | Вложения (см. [files.md](files.md)) |
| `notify` | bool | Отправить уведомление участникам чата |

**Client ID (cid):** Используется для дедупликации сообщений на клиенте.
Обычно — текущий timestamp в миллисекундах. Два сообщения с одинаковым
`cid` от одного отправителя считаются одним.

### Ответ

```json
{
  "message": {
    "id": "116762203424780659",
    "time": 17596280448116,
    "text": "Привет!",
    "sender": 3260455,
    ...
  }
}
```

Поле `message.id` может быть строкой или числом в зависимости от
транспорта.

## MSG_EDIT (opcode 67)

Редактирование отправленного сообщения.

### Запрос

```json
{
  "chatId": 7268926,
  "messageId": 116762203424780659,
  "text": "Новый текст"
}
```

### Ответ

На момент исследования опкод возвращал:
```
cmd=3
error: "Empty message can't be send"
```

Возможно, требуется дополнительный payload (например, `elements`).
Статус: **недокументирован**.

## MSG_DELETE (opcode 66)

Удаление сообщения.

### Запрос

```json
{
  "chatId": 7268926,
  "messageIds": [116762203424780659],
  "forMe": false
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatId` | int | ID чата |
| `messageIds` | array[int] | Массив ID сообщений для удаления |
| `forMe` | bool | `false` — удалить у всех, `true` — только у себя |

### Ответ

```json
{
  "chatId": 7268926,
  "messageIds": [116762203424780659]
}
```

### Особенности

- **Множественное удаление:** `messageIds` может содержать несколько ID —
  они будут удалены одним запросом
- **`forMe`:** `true` = мягкое удаление (как "удалить у себя" в Telegram).
  Сообщение исчезает только у отправителя, у других участников остаётся
- **Весь чат:** удаление служебных сообщений (типа `SERVICE`) может
  не работать или работать иначе
- **Повторное удаление:** при попытке удалить уже удалённое сообщение
  сервер возвращает `cmd=3`
- **Проверка:** единственный надёжный способ убедиться, что удаление
  сработало — запросить историю (opcode 49) и убедиться, что сообщения
  нет. Сервер отвечает `cmd=1` даже на запросы, которые не выполняют
  ожидаемого действия.

### Push-уведомления

При удалении сообщения другим участником чата приходит:
- **opcode 142** (NOTIF_MSG_DELETE) — уведомление об удалении
- **opcode 140** (NOTIF_MSG_DELETE_RANGE) — уведомление о массовом удалении

Подробнее в [push.md](push.md).

## MSG_DELETE_RANGE (opcode 92)

Массовое удаление диапазона сообщений. Статус: **не тестирован**.

Предполагаемый payload:
```json
{
  "chatId": 7268926,
  "fromId": 100,
  "toId": 200,
  "forMe": false
}
```

## MSG_TYPING (opcode 65)

Отправка индикатора набора текста. Не выполняет никаких действий
над сообщениями.

### Запрос

```json
{
  "chatId": 7268926
}
```

### Ответ

`cmd=1` — всегда успех. Это не означает, что выполнена какая-либо
операция над сообщениями. **Нельзя использовать для удаления.**

## GET_HISTORY (opcode 49)

Получение истории сообщений чата.

### Запрос

```json
{
  "chatId": 7268926,
  "forward": 0,
  "backward": 10,
  "backwardTime": 0,
  "forwardTime": 0,
  "getChat": false,
  "from": 1734567890123,
  "getMessages": true,
  "interactive": false
}
```

| Поле | Тип | Описание |
|------|-----|----------|
| `chatId` | int | ID чата |
| `backward` | int | Количество сообщений назад (до) |
| `forward` | int | Количество сообщений вперёд (после) |
| `backwardTime` | int | Сдвиг по времени назад (0 = от текущего) |
| `forwardTime` | int | Сдвиг по времени вперёд |
| `getChat` | bool | Включить информацию о чате в ответ |
| `from` | int | Timestamp, от которого начинать отсчёт |
| `getMessages` | bool | Вернуть массив сообщений |
| `interactive` | bool | Флаг для серверной оптимизации |

### Ответ

```json
{
  "messages": [
    {
      "id": 116762160362694583,
      "time": 17596280448116,
      "type": "USER",
      "sender": 3260455,
      "text": "текст",
      "attaches": [],
      "reactionInfo": {}
    }
  ],
  "chat": { ... }   // если getChat: true
}
```

### Пагинация

Для подгрузки более старых сообщений используйте `from` = timestamp
самого старого сообщения из предыдущего ответа.

### Удалённые сообщения

Удалённые сообщения **полностью отсутствуют** в ответе GET_HISTORY —
они не помечаются как `deleted`, а просто исчезают.
Это отличает MAX от Telegram, где удалённые сообщения приходят
с флагом deleted=true.

## GET_MESSAGE (opcode 71)

Получение полной информации об одном или нескольких сообщениях.

### Запрос

```json
{
  "chatId": 7268926,
  "messageIds": [116762160362694583]
}
```

### Ответ

```json
{
  "chatId": 7268926,
  "messages": [{
    "id": 116762160362694583,
    "time": 17596280448116,
    "type": "USER",
    "sender": 3260455,
    "cid": 35,
    "text": "текст",
    "attaches": [],
    "reactionInfo": {}
  }]
}
```

Полезен для отладки — показывает полную структуру сообщения,
включая поля, которые могут опускаться в GET_HISTORY.

| Операция | Опкод | Payload | cmd ответа |
|----------|-------|---------|------------|
| Отправить | 64 | `{chatId, message, notify}` | 1 |
| Удалить | 66 | `{chatId, messageIds, forMe}` | 1 |
| Удалить диапазон | 92 | `{chatId, fromId, toId, forMe}` | ? |
| Редактировать | 67 | `{chatId, messageId, text}` | 3* |
| Печатает | 65 | `{chatId}` | 1 |
| История | 49 | `{chatId, backward, from, ...}` | 1 |
| Получить сообщение | 71 | `{chatId, messageIds}` | 1 |

* — на момент исследования
